Version: 5.0.2

Release Notes

Below the list of updates in this release. Please have a look to Breaking changes with previous version of the plaform.

Updates and new features on the CLI​

Overall​

  • Speedup some operations
  • In interactive mode, the browser to select a project and/or an env is now more simple
  • Fix an issue that set interactivity in some non-interactive session

Configuration file​

  • The name of a service can be anything except /
  • Add a baseUrl configuration in service to be able to map a private port to your base URL (in replacement of the baseUrlId, which remains allowed, but deprecated). See documentation.

runner commands​

  • All runner command are now independent of the project or env selected
  • An option --path is available to specified the project or env for info and list commands. By default the list will display all runners of your organization, and the info will search the runner in all your organization.
  • In interactive mode, if there is some ambiguities in the name of a runner, a choice will be prompted to avoid adding --path option
  • Add a --output-format in model list to output in JSON, YAML or CSV
  • Display the path in info and list commands

deployment logs command​

  • Add a --service to restrict the log to a specific set of services
  • In interactive mode, the list of deployment is now sort by deployment date
  • In interactive mode, the available log levels are displayed
  • In interactive mode, all deployments without logs according to your options (--level, --since, --until, …) are grayed
  • The default log level of a deployment is set to build if there was an error during build, or all execution levels if there was no build error (aka stdout, stdin, system, access, …)

Breaking changes​

In script using runner commands, you may need to adapt them to add a --path option
