Developer Tools

We provide you development tools to code, deploy and run in the most simple way according to the service you want to use.

Note that all tools are free to use. You need a scaledynamics account to access the platform. You can sign up here: https://console.scaledynamics.com/auth/signup. Subscription is FREE, no credit card required.

SDK

Our SDK provides you a CLI you can use to deploy your assets, Node.js server or module.

npm install warp

We designed the SDK to run on desktop/laptop (windows, mac, linux) and be compatible with any IDE, bundler, static site generators, cloud-native web frameworks, CI/CDs, APIs, databases, so it's compatible with what you already use.

The SDK provides key features to deploy

  • With one command line, deploy static assets, one Node.js server or one Node.js module, or everything at once in one particular execution environment
  • Create as many projects as needed to identify your web apps, SaaS or microservices
  • Create the cloud execution environment needed for projects: development, test, staging, preproduction, production...
  • Our CLI is script friendly to be integrated it into your own CI/CD workflow using role accounts

Specifically for Managed modules our SDK provides everything to develop and emulate module execution:

  • JavaScript support
  • TypeScript is fully supported: get typed, easing the communication with team members
  • The local cloud emulation environment replicates faithfully the cloud environment of a Node.js server on laptop
  • Integrated live-reload. Change any line of client or server code and see what happens in real-time: no rebuild, no redeploy, no restart
  • Remote emulation to access remote resources, or develop for a specific device, install our emulator on a phone or on a remote server

Have a look to managed module developer workflow for more information.

Realtime playground

This is the first online playground that allows coding directly in the browser for both front-end JavaScript and back-end Node.js module, showing results in real-time with zero configuration. It provides you every you need to show front-end and back-end working together in one online place.

On the playground the backend is a standard Node.js module (not a http Node.js express server), so no need to create HTTP calls, just create a Node.js module and call its exported functions right in JavaScript.

The playground provides you a way to browse, create or fork full stack code snippets. A Snippet is composed of one HTML file, one CSS file and one JavaScript file for the browser side, and one Node.js module file with its npm module dependencies.

Each time you click on the "Run" button, the Node.js module is deployed on a real physical server on the cloud (currently on Azure). All is managed for you from the deployment or accessing the backend logs to debug your code. The preview gives you the result of your frontend + backend execution right in your browser.
