TypeScript support

In this page we explain how you can use TypeScript to build your modules.

Install TypeScript

First of all, you need to install TypeScript in your project or globally.

npm install -g typescript #globally

npm install --save-dev typescript #in your project

Configuration

You need to create a tsconfig.json file at the root of your module. You can use these options to make everything work with the SDK:

{
  "compilerOptions": {
    "target": "ES6",
    "module": "CommonJS",
    "strict": true,
    "declaration": true,
    "outDir": "dist",
    "moduleResolution": "node"
  },
  "include": ["src/**/*.ts"]
}

The target have to be set at ES6 and the module at CommonJS, for compatibility reasons.

Since the SDK work with js file and don't handle native TypeScript, you have to give it the main result file from the compilation. Since the Javascript is not strongly typed, you need to also set/define the path to the generated definition file to use them where you import your module. Open your warp.config.js file to add an expose object to give warp the js file build from TypeScript and the types (optional). For more information on expose, you can go here.

// without types
expose: 'dist/index.js'

// with types
expose: {
    source: 'dist/index.js',
    type: 'dist/index.d.ts',
  },

Finally, you can use the TypeScript compiler directly in a terminal or create npm scripts in your package.json

 "scripts": {
  "ts-watch": "tsc --watch",
  "ts-build": "tsc"
 }

After that, you should be able to use your fully typed module wherever you import it.

Using JS and JSDoc to generate types

If you want to make a module in JS but use it in a TypeScript environnement (an Angular frontend for example) and want to use types, you can write JSDoc and use the TypeScript compiler to generate them. Change the tsconfig.json options to :

{
  "compilerOptions": {
    "strict": true,
    "allowJs": true,
    "declaration": true,
    "emitDeclarationOnly": true,
    "outDir": "dist",
    "moduleResolution": "node"
  },
  "include": ["src/**/*.js"]
}

TypeScript will only generate the types files from the jsdoc. You can add the main type file on your warp.config.js file. This TypeScript declaration file can import any other declaration file.

expose: {
    source: 'src/index.js',
    type: 'dist/index.d.ts',
  },

With this solution, you can't have live reload on the types. Every time you update your JSDoc, you need to launch the TypeScript compiler to build the declarations files.

tsc build
