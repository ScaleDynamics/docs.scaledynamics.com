Glossary

Our computing platform defines some notions: user, organization, project, environment and role account.

User

A user is identified with an email uniq address. A User can be part of several Organizations.

Organization

An organization identifies a set of users and rights. An organization has several projects and can create several role accounts. Each organization as a product plan.

Project

A project identifies a web site, a web app, a microservice, an API... Users of the organization can be assigned to projects with specific rights.

Environment

An environment identifies a cloud execution configuration to run modules

Cloud resource

A physical cloud resource configuration is used to run physically modules. In can be attached to one environment, or to an organization. In this case it will be used to run all environments of the organization that don't have dedicated cloud resources.

Role Account

Role accounts can be used by CI/CD to automatize test workflows or deployments.
