Platform features

Cloud emulator

We provide you a cloud emulator you can use to code/test your modules with live reload.

More details on the emulator

Deployment and setup

With our platform, deploy your module in one command line.

At deployment step, we manage the setup of the endpoint, creation of HTTPS certificates, docker build, ... Nothing to take care on your side!

More details on the deployment

Provisioning

From the cloud store you can provision a cloud resource to run your module in few clicks.

A provisioning can be done per environment (dedicated to run only the environment) or per organization (used to run all environments of the organization which don't have a dedicated resource).

Browse it.

Load balancing

The load balancing share requests execution using all the available virtual CPUs of the cloud resource.

To support more traffic, just select a resource with more VCPUs. Decrease the number of VCPUs if your traffic decreases.

Transparent automatic retry

This feature is available only when using direct module calls. Behind the scene, our compiler generates all the http layers to manage module calls. If an HTTP request fails, we manage an automatic retry according to the cloud resource type (provider, network...), taking care of the number of retries and a timeout transparently for you.

Log access

To get the execution logs, use our CLI to access logs of the module on the cloud resource. You can also to a live refresh log update to monitor dynamically the execution

Look at the cli

Complementary static hosting

We provide a basic hosting of static assets. Create URLs on our ScaleDynamics test domains, or use your own custom domains.

More details on the deployment of static assets

ScaleDynamics cloud

Run on our best effort cloud for testing the platform.

It's a no-warranty cloud with low-processing configuration.

Use your cloud resources to get a warranty your modules will run.
