Getting started

On our platform you can deploy a standard http Node.js server (i.e. using Express.js) so you can access it with traditional http requests . You can use your favorite Node.js web framework to host all all your website by serving your static assets and backend, your microservice APIs, a backend proxy... without managing the cloud itself (provisioning, deployment, setup, HTTPS certificates, security patches, cloud change...)

In this quick tutorial, you'll learn how to create and deploy a basic Express.js http Node.js server you can call from http.

1. Prepare the directory

Let's create a my-server working directory.

mkdir my-server
cd my-server/

2. Create a basic http Express.js Node.js server

Create a package.json file

{
  "name": "my-server",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  }
}

Install Express.js

npm install express

Create index.js

Create a new index.js file in the server directory, and copy-paste the following code into it:

// my-server/index.js

const express = require('express');
const app = express();
const port = process.env.WARP_PORT ? process.env.WARP_PORT : 3000

app.get('/', (req, res) => {
  res.send(`Hello world from Node.js ${process.version}`)
});

app.listen(port, () => {
  console.log(`This app listens on the port ${port}`);
});

3. Install our project SDK

To install the SDK you need Node.js installed on your computer. Look at https://nodejs.org/en/download/ to install it.

Install the SDK to access all CLI commands:

npm install warp

To access our CLI, use npx warp. You can have the list of available commands and help with

npx warp help

4. Deploy the server

The deployment of a server module requires to indicate in which project and which environment you want to deploy and execute your module.

A project is a name that identifies a web site, a web app, a microservice or an API.

An environment is a name that identifies a cloud execution configuration to run modules. For example you can have 'pre-prod', 'demo', 'staging' or 'production' environments. Each one will have it's own cloud resource configuration.

Login to your account

To access projects and deployment resources you need to login to your ScaleDynamics account:

npx warp login

Note that if you don't have a ScaleDynamics account, you can sign up here: https://console.scaledynamics.com/auth/signup. Subscription is FREE, no credit card required.

Create a project

A project identifies a web site, a web app, a microservice or an API.

Let's create a server-test project, and let's indicate the SDK we are working in it.

npx warp project create server-test
npx warp project select server-test

Create an environment

Let's create a demo environment, and let's indicate the SDK we are working in it.

npx warp env create demo
npx warp env select demo

Deploy the server

We can deploy the server module.

npx warp deploy

During the deployment, you will have to indicate the url where you want to access your server after deployment.

✔ Enter a hostname for 'server' (fully qualified or not), leave blank for random: …

You can enter a name or press return to get a random one.

After deployment the url to use to call your module is dumped on the terminal.

if you want to know the deployment url, you can access the console or use the following command to get it:

npx warp env info

5. Call the server

Now the server is deployed, let's call it from http using curl for example:

curl https://DEPLOYMENT_URL
