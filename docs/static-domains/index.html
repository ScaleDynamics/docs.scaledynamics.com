Use custom domains

The platform provides you test urls running on scaledynamics.site, or scaledynamics.cloud.

You can easily connect your custom domains to make your assets live at www.mycie.com for example.

1. Create a new deployment environment

Before deploying, you need to create a new deployment environment, and start using it. Of course you need to have selected a project before.

For example let's create a prod env, you can use the environment name you want.

In your asset directory:

npx warp env create prod
npx warp env select prod

2. Assign your custom domain for deployment

Then we will connect the url of your deployment to that environment. In our case we want to use www.mycie.com

npx warp env base-url add hosting www.mycie.com

The command will dump something like that

Base URL ID 'hosting' with hostname 'www.yourcompany.com' added.
Please deploy a DNS CNAME record under the name 'www.yourcompany.com' with value 'xxxxxxxxxxxxx.scaledynamics.site.' before deploying

Follow the guideline and add a DNS CNAME record with the value provided.

3. deploy

Now you can deploy as usual.

npx warp deploy

After deployment, the command will dump your url https://www.yourcompany.com you can access your assets.
